git rebase를 사용하자

일단 최종커밋을 한다

그러고나서 바꿀 총 커밋의 개수를 센다

만약 이전에 2개가있고 방금 새로 1번의 커밋을 더했다면 직전 커밋은 세개가 되니까

git rebase -i HEAD~3 이렇게 하면 vi 창이 하나 뜬다 

pick asd
pick asd
pick asd

이런식으로 되어있는데 이 pick 부분을 fixup으로 바꿔준다 두개를 바꿔준다

그럼 세개의 커밋이 하나로 된다

진짜 방법 합칠 커밋은 pick 고 바꿔줄 부분을 squash 로 바꿔주거나 약어인 s로 바꿔주면

커밋 vi 창이뜬다 여기서 커밋메세지 수정해주면 깔끔하게 하나로 통합(바꾸려고헀던 모든 커밋들이 합쳐지며 새로운 커밋을 생성하고 이전 로그들은 싹지워짐)



그러고나서 git commit --amend로 커밋메세지 바꿔주면 끝!

reflog는 모든 작업이 다나오는거고 log는 커밋정보만 나오는거같다
